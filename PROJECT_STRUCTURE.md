# Project Structure

```
src/
├── frontend/              ← UI: routes, components, hooks, lib (auth/cart/i18n), styles, assets
│   ├── routes/            ← TanStack Router file-based routes (URLs / pages)
│   ├── components/        ← Header, Footer, ProductCard, shadcn UI
│   ├── hooks/
│   ├── lib/
│   ├── router.tsx         ← createRouter + route tree import
│   ├── routeTree.gen.ts   ← generated by TanStack Router (do not edit)
│   └── styles.css

├── backend/               ← Server/data boundary (re-exports + server helpers)
│   ├── db/                ← Supabase clients & types (see supabase/typescript)
│   └── server/            ← TanStack server functions (createServerFn)

supabase/
├── migrations/            ← SQL schema, RLS, triggers
├── config.toml
└── typescript/            ← Supabase JS clients & generated DB types (browser + server)

```

## Imports

- App code imports UI from `@/frontend/...` (e.g. `@/frontend/components/...`, `@/frontend/lib/...`).
- Data access from the browser uses `@/backend/db/client` (publishable key + RLS).
- Server-only admin client: `@/backend/db/client.server` or `@supabase/client.server` (never expose service role to the client bundle).

If you use **Lovable Cloud** regenerating Supabase files, sync regenerated output into `supabase/typescript/` (previously `src/integrations/supabase/`).
